# Cas d'usage IA

Les cas d'usage sont des opportunités validées et spécifiées avec un périmètre technique et un ROI estimé.

## Qu'est-ce qu'un cas d'usage ?

Un cas d'usage représente une opportunité IA qui a été :
- **Qualifiée** : faisabilité technique confirmée
- **Spécifiée** : périmètre, données et contraintes définis
- **Valorisée** : ROI estimé avec hypothèses documentées

## Créer un cas d'usage

1. Partez d'une opportunité qualifiée dans le pipeline
2. Cliquez sur **"Convertir en cas d'usage"**
3. Complétez la fiche technique : données requises, modèle envisagé, intégrations
4. Renseignez le business case : gains attendus, coûts estimés, délai de retour

## Cycle de validation

| Étape | Description |
|-------|-------------|
| Brouillon | En cours de spécification |
| En révision | Soumis pour validation métier et IT |
| Validé | Approuvé et prêt pour la roadmap |
| En production | Déployé et en cours de mesure |

## Suivre les résultats

Une fois un cas d'usage en production, renseignez les **résultats réels** pour comparer avec les estimations initiales. Ces données alimentent le ROI Dashboard et le briefing hebdomadaire.

## Bonnes pratiques

- Un cas d'usage = un problème précis, pas un projet global
- Documentez les **hypothèses** de ROI pour faciliter la révision
- Associez chaque cas d'usage à un [processus métier](/app/processes) et un [domaine](/app/domains)
